Flex Container

when we set the display property of the container to flex, the elements will be aligned side by side as shown:

.container { background-color: #ccc; padding: 10px; display: flex; }

flex-direction

The default value for the flex-direction is row, and that is why the elements are side by side.

row-reverse:

.container { background-color: #ccc; padding: 10px; display: flex; flex-direction: row-reverse; }

When we set the flex-direction property to be column, the main axis now goes from top to bottom, and the items end up stacking on top of each other:

.container { background-color: #ccc; padding: 10px; display: flex; flex-direction: column; }

Using column-reverse makes the main axis go from bottom to top, and the items end up as shown below:

justify-content

center: all the items are aligned to the center of the main axis.

.container { display: flex; justify-content: center; }

space-between: the available space is evenly distributed between the flex items.

.container { display: flex; justify-content: space-between; }

space-around: puts the same amount of space on both the left side and right side of each flex item.

.container { display: flex; justify-content: space-around; }

experiment with space-evenly, flex-end and flex-start which is the default

align-items

To better visualize the working of align-items property, let's change the height of one of the flex items.

<div class="container"> <div class="item">1</div> <div class="item i2">2</div> <div class="item">3</div> <div class="item">4</div> <div class="item">5</div> </div>
.i2 { height: 200px; }

Notice that all the elements end up getting the 200px height, this is because the initial value on align-items property is stretch.

center: center flex-items relative to the item which is taller/bigger in the vertical direction.

.container { display: flex; justify-content: center; align-items: center; }

flex-start

.container { display: flex; justify-content: center; align-items: flex-start; }

experiment with the flex-end value and baseline as well
